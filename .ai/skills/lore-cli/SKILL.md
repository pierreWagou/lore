# lore CLI

lore is an agent skills package manager. It fetches skills from any git source and installs them in each harness's native format. `.ai/skills/` is the project-local neutral store per the [agentskills.io](https://agentskills.io) standard; `lore.toml` and `lore.lock` track what is installed.

## Key concepts

| Concept | Description |
|---|---|
| `lore.toml` | Manifest declaring skill dependencies and target harnesses |
| `lore.lock` | Lockfile pinning exact commit SHAs and content hashes — commit this |
| `.ai/skills/<name>/` | Neutral local skill store, committed to git |
| `.opencode/skills/`, `.claude/skills/` | Generated by lore (symlinks/copies), gitignored |
| Harnesses | opencode, claude, cursor, codex — each with their own native format |

## Commands

### lore init

Create `lore.toml` and update `.gitignore` with harness dirs. Detects installed harnesses automatically.

```bash
lore init          # project-scoped (./lore.toml)
lore init -g       # global (~/.config/lore/lore.toml)
```

### lore add

Fetch a skill from a git source and install it.

```bash
# Project install — added to lore.toml, source in .ai/skills/
lore add owner/repo/path/to/skill
lore add owner/repo/path --ref v1.2.0          # pin to tag
lore add owner/repo/path --harness opencode     # specific harness only

# Global install — no lore.toml entry, goes directly to harness global dirs
lore add -g owner/repo/path/to/skill

# Scan whole repo (prompts to select)
lore add owner/repo
lore add owner/repo --all                      # install all without prompting
```

### lore create

Scaffold a new project-local skill in `.ai/skills/<name>/SKILL.md`.

```bash
lore create my-skill
# → creates .ai/skills/my-skill/SKILL.md with frontmatter template
# → adds to lore.toml
# → creates symlinks in harness dirs
```

Edit the skill, then run `lore sync` to propagate changes to harnesses that use copies.

### lore sync

Install all skills from `lore.toml`. This is the setup step after cloning a repo.

```bash
lore sync          # project skills from ./lore.toml
lore sync -g       # global skills from ~/.config/lore/lore.toml
```

### lore remove

Remove a skill from the manifest, neutral store, and all harness dirs.

```bash
lore remove my-skill
lore remove -g my-skill
```

### lore list

Show installed skills with source, ref, and pinned commit.

```bash
lore list
lore list -g
```

### lore harnesses

Detect which harnesses are installed on the current machine.

```bash
lore harnesses
```

### lore auth

Store authentication tokens for private git hosts.

```bash
lore auth add github.com ghp_yourtoken
lore auth add gitlab.com glpat_yourtoken
lore auth list
lore auth remove github.com
```

## Source handle formats

```
owner/repo/path/to/skill          GitHub shorthand — explicit path
owner/repo                        GitHub shorthand — scan whole repo
https://github.com/owner/repo/tree/main/path   Full HTTPS URL
git@github.com:owner/repo.git/path             SSH (private repos)
./local/path/to/skill             Local filesystem (relative)
/absolute/path/to/skill           Local filesystem (absolute)
```

## Project skill workflow

```bash
# 1. Initialise (once per project)
lore init

# 2. Create a project skill
lore create my-skill
$EDITOR .ai/skills/my-skill/SKILL.md

# 3. Sync after edits (needed for harnesses that use copies, e.g. cursor)
lore sync

# 4. Commit source — not harness dirs
git add .ai/skills/ lore.toml lore.lock
git commit -m "feat: add my-skill"

# 5. After clone, restore harness symlinks
lore sync
```

## Authentication

HTTPS token resolution order (first match wins):

1. `LORE_GITHUB_COM_TOKEN` env var (host-specific override)
2. `GITHUB_TOKEN` or `GH_TOKEN` env var
3. `gh auth token` (gh CLI, if installed)
4. `~/.config/gh/hosts.yml` (gh CLI fallback)
5. Token stored via `lore auth add github.com <token>`

For other hosts (`gitlab.com`, self-hosted):

1. `LORE_<HOST>_TOKEN` env var (dots and dashes replaced with `_`, uppercased)
2. Token stored via `lore auth add <host> <token>`

SSH: uses system SSH agent (`SSH_AUTH_SOCK`) then auto-detects `~/.ssh/id_*` key files.

## lore.toml reference

```toml
# Which harnesses to install skills into.
# Overridden by --harness flag. Auto-detected if omitted.
harnesses = ["opencode", "claude"]

[[dependencies]]
name   = "lore-cli"                              # unique identifier
source = "pierreWagou/lore/lore-cli"             # any source handle format
ref    = "main"                                  # branch, tag, or full SHA

[[dependencies]]
name   = "pdf"
source = "git@github.com:my-org/skills.git/pdf"  # SSH for private repos
ref    = "a1b2c3d4e5f6789012345678901234567890"  # pin to SHA for production

[[dependencies]]
name   = "local-skill"
source = ".ai/skills/local-skill"                # project-local skill
ref    = ""
```

## Installing this skill

```bash
# Install lore-cli skill globally
lore add -g pierreWagou/lore/lore-cli

# Install for a project
lore add pierreWagou/lore/lore-cli
```
